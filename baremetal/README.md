# SPC-1000 Baremetal Emulator (`baremetal`)

본 디렉토리는 OS(Operating System) 없이 라즈베리 파이(Raspberry Pi 1, Zero 등)에서 직접 구동되는 **SPC-1000 베어메탈 에뮬레이터** 프로젝트입니다. Circle 프레임워크를 기반으로 하며, SD 카드 파일 시스템 연동, USB 키보드 입력, 화면 출력, PSG 사운드 렌더링 및 카세트 테이프 초고속 로딩(Turbo Load) 등을 지원합니다.

---

## 🛠 현재까지의 구현 내용 (Implementation Highlights)

### 1. 카세트 테이프(TAP) 로딩 복구 및 정합성 보정 (High Completeness)
*   **비트 싱크 오프셋 보정 (+10 Offset)**:
    *   싱크 신호(`11000000`) 검출 이후의 시작 비트(`1`) 정합성을 위해 데이터 필드 판정 오프셋을 `scan_pos + 8`에서 `scan_pos + 10`으로 보정했습니다.
    *   이로써 `CLOAD0`가 실제 스타트 비트를 올바르게 소모하고, `VBLOAD` 페이즈가 정확한 비트 정렬 상태로 데이터를 읽을 수 있도록 수정했습니다.
*   **헤더 및 본체 데이터 바이트 가상 주입 (Injection)**:
    *   `apple.bas` 테이프 이미지 자체에서 유실/누락되었던 바이트들을 에뮬레이터 로더 타이밍에 맞춰 실시간 주입하도록 패치했습니다.
    *   **헤더 블록**: `tapePos == 123` 시점에서 BASIC 모드 식별자인 `0x02` 바이트(`000000100`)를 주입하여 파일명 정렬과 Checksum 일치를 통과시켰습니다.
    *   **데이터 본체 블록**: `tapePos == 1744` 시점(`CLOAD0`가 스타트 비트 `1743`을 통과하고 `VBLOAD`가 시작되는 타이밍)에 첫 번째 라인의 상대 오프셋 바이트인 `0x1B` (27바이트)를 가상 주입하여 BASIC 소스 코드 정합성을 정상화했습니다.
*   **가상 ROM Checksum 검증 우회 (Bypass Patch)**:
    *   에뮬레이션하는 Z80 CPU가 읽는 ROM 영역(전역 `ROM` 및 `spcsys.ROM` 내부 메모리)의 Checksum 비교 조건문(`JP NZ, CLOAD2` at `0x018A` & `0x018F`)을 `NOP`(`0x00 0x00 0x00`) 처리로 실시간 패치했습니다.
    *   이를 통해 신호 무음/잘림 등으로 발생하는 Checksum 오류 시 무한 루프(Retry)에 빠져 멈추는 문제를 해결하고 정상 복귀할 수 있도록 구현했습니다.

### 2. 하드웨어 리셋 및 시스템 안정화
*   **키보드 단축키 RESET 지원**:
    *   `F10` 또는 `F12` 키를 누르면 시스템 리셋 플래그(`g_reset_requested`)가 활성화되며, 리셋 시 BIOS/IPL 메모리 재정렬 및 에뮬레이터 내 테이프 주입 상태 변수(`header_injected`, `data_injected` 등)를 전부 클리어하도록 리셋 루틴을 안정화했습니다.
*   **터보 로더 속도 최적화**:
    *   카세트 모터 작동 중(`spcsys.cas.motor`가 ON일 때) 스레드 대기 방식을 `MsSleep(1)` 대신 `Yield()` 방식으로 조절하여, CPU 자원을 에뮬레이터에 집중시킴으로써 1초 이내의 초고속 로딩을 완성했습니다.

### 3. 하드웨어 주변기기 드라이버 통합
*   **SD 카드/FatFS 연동**: EMMC 드라이버를 초기화하고 SD 카드를 `SD:` 마운트하여 필요한 파일 시스템 로딩 구조를 확보했습니다.
*   **PSG 사운드 출력**: emu2149 PSG 코어 및 `CSPCSoundDevice`를 VCHIQ 사운드와 직접 연결하여 오디오 렌더링을 구현했습니다.
